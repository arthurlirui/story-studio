# 《重生穿越》系列 — 10部中篇小说执行计划

> 每部：8章 × ~1万字/章 = ~8万字 | 总计：80万字
> 核心主题：宿命论 — 无论主角如何操作，历史终将走向已知的结局

---

## 一、项目总览

### 1.1 执行策略

使用 story-studio 编排器，按 Idea 1→10 顺序逐部推进。每部走完整 pipeline：

```
策划(Planning) → 建立(Building) → 大纲(Outlining) → 写作(Writing) → 完稿(Complete)
```

### 1.2 每部工作流

| 阶段 | 内容 | 预估轮次 |
|------|------|---------|
| **Planning** | Showrunner 分析 Idea → 团队讨论 → 输出创作企划书 | 3轮 API |
| **Building** | 世界观架构师细化时代设定 → 角色设计师创建核心角色（3-5人） | 3轮 API |
| **Outlining** | Showrunner 生成8章大纲 → 文学顾问审阅 → 终版大纲 | 3轮 API |
| **Writing** | 8章逐章写作（可并行2-3章）→ 编辑润色 → 连续性检查 | 8×3=24轮 API |
| **Complete** | 全文终审 → 输出最终稿 | 3轮 API |
| **合计** | 每部约 36 轮 API 调用 | |

### 1.3 时间估算

- 每部中篇：约 2-3 天（含人工审阅和修订）
- 10部总计：约 20-30 天
- 建议节奏：每完成2部暂停，回顾质量，调整策略

## 三、宿命论统一框架

### 3.1 每部小说的宿命结构

```
第1章：穿越觉醒 —— 主角来到古代，确认时代，知道"历史结局"
第2章：确认锚点 —— 深入理解所处时代，发现"历史的齿轮已经开始转动"
第3章：第一次干预 —— 尝试用现代知识做点什么（初心：改善生活/帮助他人）
第4章：蝴蝶效应 —— 干预产生意外后果，事情开始偏离预期
第5章：修正尝试 —— 发现不对劲，试图"纠正"自己的影响
第6章：历史的陷阱 —— 修正行为反而加速了历史进程，越挣扎陷越深
第7章：宿命揭示 —— 意识到自己的所有行为都是历史必然的一部分
第8章：接受/超越 —— 在宿命中找到个人意义，完成角色弧光
```

### 3.2 宿命论的五个层次

1. **个人宿命**：主角个人的命运不可逃避
2. **事件宿命**：具体历史事件的结果不可改变
3. **结构宿命**：社会结构和历史规律不可违抗
4. **因果宿命**：主角的行为本身就是历史结果的原因
5. **意义宿命**：虽然结果不变，但过程赋予了新的意义

### 3.3 每部小说的宿命核心

| # | 主角想改变/回避什么 | 如何阴差阳错导致同样结果 | 宿命层次 |
|---|-------------------|----------------------|---------|
| 1 | 不想参与赤壁之战 | 武器技术被曹操获取→过度自信→同样被火攻击败 | 因果宿命 |
| 2 | 想提前疏散避免崖山 | 疏散引发恐慌→防线崩溃→败得更快 | 事件宿命 |
| 3 | 想改革挽救大明 | 改革触动利益→加速内乱→李自成更快入京 | 结构宿命 |
| 4 | 只想经商远离政治 | 商业网络被安禄山利用→叛乱规模更大 | 因果宿命 |
| 5 | 想帮项羽改变历史 | 帮助让项羽更傲慢→失去人心→仍败于刘邦 | 个人宿命 |
| 6 | 想用现代医学救人 | 救活的人变成士兵→死于战争→死亡总数不变 | 结构宿命 |
| 7 | 想建立制度防屠戮 | 制度引发猜忌→屠杀来得更早更狠 | 因果宿命 |
| 8 | 想强化国防御金 | 军备刺激金国→入侵提前→靖康之变更惨 | 事件宿命 |
| 9 | 想开放海禁兴海洋 | 海上势力引朝廷恐惧→海禁更严 | 结构宿命 |
| 10 | 只想种地不问世事 | 良田成战略要地→卷入安史之乱 | 个人宿命 |

## 四、质量保障

### 4.1 每部完成后检查清单

- [ ] 宿命论主题是否贯穿始终
- [ ] 主角的"不愿改变历史"动机是否合理
- [ ] 历史走向的"阴差阳错"是否自然（非机械降神）
- [ ] 角色弧光是否完整
- [ ] 时代细节是否准确
- [ ] 现代知识的应用是否合理（有前置条件约束）
- [ ] 8章节奏是否紧凑（每章有独立钩子）

### 4.2 跨作品一致性

- 穿越机制不冲突（不同作品可用不同方式）
- 历史事件的描述一致（同一事件在不同作品中不矛盾）
- 知识金手指的难度等级统一
