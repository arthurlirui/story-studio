# 哥伦布计划 · 编写经验 / 踩坑笔记 (Lessons)

> 自动编写流程踩过的坑，写在这里，别再犯。

## 2026-07-04 — edit 大段追加导致 run 报错、正文偏短

**现象**：cron 自动编写第 02 本时连续 2 次 run 报 error，报错都是
`📝 Edit: in .../02_.../full.md failed`。第 02 本最终只有 ~1.9 万字（目标 3 万），偏短。

**根因**：编写 agent 习惯"先 write 写开头，再用 edit 逐章追加正文"。大段文本用 edit 追加时，
`oldText` 很难精确唯一匹配，导致 edit 失败、后续章节没能写进去，字数也就上不去。

**教训 / 规则**：
- 写整篇小说**一次性用 write** 写完整个 `full.md`（书名+简介+6章全部正文）。
- 正文太长一次写不下时，**按章分别 write** 到 `ch1.md … ch6.md`，最后 `cat ch*.md > full.md` 合并。
- **绝不用 edit 做大段正文追加**。edit 只适合小范围、能唯一匹配的精确修改。

**已落地**：cron「哥伦布计划-小说自动编写」的 payload 提示已加入上述「写入方式」硬约束。

**附带决定**：字数不强求。目标约 3 万字为参考，写完整、写好看优先，不为凑数注水。
